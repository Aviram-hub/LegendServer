# 数据库设计文档

## 数据库架构

### 分库策略

- 按玩家ID取模分库
- 日志库独立部署
- 配置库独立部署

### 分表策略

- 日志表按日期分表
- 大表按玩家ID哈希分表

## 表结构设计

### 核心表

1. **player_account** - 玩家账号表
2. **player_role** - 角色表
3. **role_attribute** - 角色属性表
4. **item** - 物品表
5. **role_skill** - 技能表
6. **friend_relation** - 好友关系表
7. **chat_log** - 聊天记录表
8. **team** - 组队表
9. **team_member** - 队员表

## 索引设计

### 主键索引

所有表使用自增BIGINT作为主键

### 唯一索引

- 账号名唯一
- 角色名唯一
- 账号+服务器唯一

### 普通索引

- 外键字段建立索引
- 时间字段建立索引
- 查询热点字段建立索引

## 缓存设计

### Redis缓存策略

| 数据类型 | 过期时间 | 说明 |
|---------|---------|------|
| Session Token | 24小时 | 玩家会话 |
| 角色信息 | 1小时 | 热点数据 |
| 在线状态 | 5分钟 | 心跳刷新 |
| 排行榜 | 实时 | Sorted Set |

### 缓存更新策略

- 读穿透：先读缓存，未命中读数据库并回填
- 写穿透：先写数据库，再更新缓存
- 延迟双删：高一致性场景使用

## 数据迁移

### 在线迁移方案

1. 双写新旧库
2. 历史数据迁移
3. 校验数据一致性
4. 切换读流量

## 备份策略

- 主从复制
- 每日全量备份
- 实时binlog备份
