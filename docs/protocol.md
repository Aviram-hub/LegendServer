# 协议设计文档

## 协议格式

### 消息结构

```
┌─────────────────────────────────────────────────────────────┐
│  协议头 (Header) - 13字节                                    │
├──────────┬──────────┬──────────┬──────────────────────────┤
│ 魔数(4B)  │ 版本(1B) │ 长度(4B) │ 消息ID(4B)               │
├──────────┴──────────┴──────────┴──────────────────────────┤
│  协议体 (Body)                                               │
├──────────────────────────────────────────────────────────┤
│  Protobuf序列化数据                                          │
└──────────────────────────────────────────────────────────┘
```

### 字段说明

| 字段 | 长度 | 说明 |
|------|------|------|
| 魔数 | 4字节 | 固定值0x4C475356 ("LGSV") |
| 版本 | 1字节 | 协议版本，当前为1 |
| 长度 | 4字节 | 整个消息长度(含头部) |
| 消息ID | 4字节 | 消息类型标识 |
| 协议体 | 变长 | Protobuf序列化的消息体 |

## 编解码

### 编码流程

1. 序列化Protobuf消息体
2. 计算总长度
3. 填充协议头
4. 发送数据

### 解码流程

1. 读取至少13字节头部
2. 校验魔数和版本
3. 读取完整消息
4. 反序列化Protobuf

## 加密

- 使用AES加密消息体
- 密钥通过登录流程协商
- 每个会话使用不同密钥
